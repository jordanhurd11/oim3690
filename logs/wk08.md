Iteration 1 -
What does my API response look like?
- JSON with elements array — one entry per golf course
- Each has type (node/way/relation), id, tags object
- Tag coverage inconsistent — some courses just name, others 20+ fields

What fields did I use?
- tags.name, tags["addr:street"], tags["addr:city"]
- type + id (for map link)
- lat/lon or center.lat/center.lon

What was the trickest part of getting fetch to work?

Overpass uses POST with a query languages string body not the typical GET = parameters

If AI helped, what did it explain about async/await?

- fetch() returns a Promise immediately, not the data itself
- await pauses the function until it resolves — only works inside async functions
- response.json() is also async, so you await twice (fetch, then parse)

Iteration 3
- What extension did I add? 
Added ability to favorite courses, added distance/ sort by, added near me feature

- What edge cases did I handle? 

Empty search input → "Type a city or region to search"
Search with no results → distinct message, no broken layout
API rate limit (HTTP 429) → distinct message + "Try again" button — confirmed against a real 429 from Overpass during testing
Generic API failure (504/network error) → "Try again" button reusing the last search — also confirmed against a real 504 live
Removing the last favorite while browsing favorites → empty-state message reappears instead of a blank grid (this was a real bug I caught and fixed during testing)
Courses missing coordinates → sort to the end instead of showing a fake/broken distance
Geolocation denied or unsupported → distinct friendly message, doesn't crash
Duplicate recent searches → moves existing entry to front instead of listing it twice
Rapid overlapping searches (stale results race) → each search is stamped with a request ID so an older, slower response can't overwrite a newer one's results

- What surprised me about working with a real API?

It getting rate limited when I am testing my code changes. Cons of using a free api
